1. Transitions

What does the W3C say? 🧐

CSS transitions allow you to change property values smoothly, over a given duration.

In simple words: it's like an easy to implement animation, which we'll take a look at later on.


How can we use them?

Let's take a look at the main properties that take part in a transition:

1. Transition Property

Here, we specify WHAT we want to animate. For example, if we want to animate the background color, we'd write:

.element {
    transition-property: background-color;
}

2. Transition Duration

Here, we specify HOW LONG we want to animate it. For example, if we want the animation to last 0.5 seconds, we'd write:

.element {
    transition-duration: 0.5s;
}

3. Transition Timing Function

Here, we specify HOW we want to animate it. For example, if we want the animation to be smooth at the beginning and end, we'd write:

.element {
    transition-timing-function: ease-in-out;
}

There are several options here, and they all affect the timing of the transition. You can check them out here.


There's also Transition Delay, but I'm sure you are smart enough to know what it does 😜


Now, let's compact it all!

Instead of writing all of these properties separately, we can write them all in one line:

.element {
    transition: background-color 0.5s ease-in-out;
}

On to some examples!

Let's take a look at some examples of transitions:

How will the previous example look like if applied to a cool button? 
background-color: blue;
transition: background-color 0.5s ease-in-out;
&:hover {
    background-color: yellow;
}

Maybe something a bit spookier?

transition: opacity 0.5s ease-in-out;
&:hover {
    opacity: 0;
}
👻

Or maybe something a bit more complicated? Let's see what else can we do with transitions!

transition-property: width, height, background-position, color, font-size;
transition-duration: 1.5s;
transition-timing-function: ease-in-out;
👉
Hello!

As you can see, here we have animated the width, height, background position, color, and font size of the element. To do that, we divided the property transition into its different parts, but we could have written it all in one line: 

transition: width 1.5s ease-in-out, height 1.5s ease-in-out, background-position 1.5s ease-in-out, ...;

When do we use them?

Transitions are great for simple animations, like hover effects, or for when you want to animate a single property. They are easy to implement and don't require much code.

Let's imagine our amazing design teams creates a new component with several states:

Main Hover Active Disabled

Let's try to create said button without transitions:

.ex-btn {
        background-color: blue;
    }
    .ex-btn:hover {
        background-color: darkblue;
    }
    .ex-btn:active {
        background-color: yellow;
    }
    .ex-btn:disabled {
        background-color: gray;
    }

We can see the changes are abrupt and not very appealing. Now, let's try to create it with transitions:

.ex-btn {
        background-color: blue;
        transition: background-color 0.25s ease-in-out;
    }
    .ex-btn:hover {
        background-color: darkblue;
    }
    .ex-btn:active {
        background-color: yellow;
    }
    .ex-btn:disabled {
        background-color: gray;
    }

Much better, right? The transitions make the changes smoother and more appealing to the users.

Exercises

If you want to practice a bit by yourself, why don't you try to create some of the following:

  • A link that slowly changes opacity when disabled.
  • A button that slightly grows larger when hovered.
  • A div with a gradient background that slowly changes its background position when clicked.

2. Animations

What does the W3C say? 🧐

CSS allows animation of HTML elements without using JavaScript!.

Let us ponder in that affirmation for a second here. Back in the day -I guess, I wasn't around really- if a developer wanted to animate an element, they'd have to use JavaScript. But now, thanks to the advancements in CSS, we can animate elements without having to write a single line of JS code. This has several implications, but the most important one is that these animations are pre-calculated by the browser, which makes them more efficient. This comes with a cost, though: CSS animations are less flexible than JavaScript animations, but there are ways to work around that.


How can we use them?

Let's take a look at the main properties that take part in an animation. We can distinguish two main parts: Keyframes and Animation Properties.

1. Keyframes

Keyframes are the steps of the animation. They define the start and end of the animation, as well as any intermediate steps. We can define as many keyframes as we want, but the most common ones are 0% and 100%, which represent the start and end of the animation, respectively. Let's see an example: 

@keyframes example-1 {
    0% {
        background-color: yellow;
    }
    100% {
        background-color: orange;
    }
}

In this example, we have defined a keyframe of name example-1 that changes the background color of the element from yellow to orange. Naming your animations is essential, as it's the way you'll reference them later on.
