1. Transitions

What does the W3C say? 🧐

CSS transitions allow you to change property values smoothly, over a given duration.

In simple words: it's like an easy to implement animation, which we'll take a look at later on.


How can we use them?

Let's take a look at the main properties that take part in a transition:

1. Transition Property

Here, we specify WHAT we want to animate. For example, if we want to animate the background color, we'd write:

.element {
    transition-property: background-color;
}

2. Transition Duration

Here, we specify HOW LONG we want to animate it. For example, if we want the animation to last 0.5 seconds, we'd write:

.element {
    transition-duration: 0.5s;
}

3. Transition Timing Function

Here, we specify HOW we want to animate it. For example, if we want the animation to be smooth at the beginning and end, we'd write:

.element {
    transition-timing-function: ease-in-out;
}

There are several options here, and they all affect the timing of the transition. You can check them out here.


There's also Transition Delay, but I'm sure you are smart enough to know what it does 😜


Now, let's compact it all!

Instead of writing all of these properties separately, we can write them all in one line:

.element {
    transition: background-color 0.5s ease-in-out;
}

On to some examples!

Let's take a look at some examples of transitions:

How will the previous example look like if applied to a cool button? 
background-color: blue;
transition: background-color 0.5s ease-in-out;
&:hover {
    background-color: yellow;
}

Maybe something a bit spookier?

transition: opacity 0.5s ease-in-out;
&:hover {
    opacity: 0;
}
👻

Or maybe something a bit more complicated? Let's see what else can we do with transitions!

transition-property: width, height, background-position, color, font-size;
transition-duration: 1.5s;
transition-timing-function: ease-in-out;
👉
Hello!

As you can see, here we have animated the width, height, background position, color, and font size of the element. To do that, we divided the property transition into its different parts, but we could have written it all in one line: 

transition: width 1.5s ease-in-out, height 1.5s ease-in-out, background-position 1.5s ease-in-out, ...;

When do we use them?

Transitions are great for simple animations, like hover effects, or for when you want to animate a single property. They are easy to implement and don't require much code.

Let's imagine our amazing design teams creates a new component with several states:

Main Hover Active Disabled

Let's try to create said button without transitions:

.ex-btn {
    background-color: blue;
}
.ex-btn:hover {
    background-color: darkblue;
}
.ex-btn:active {
    background-color: yellow;
}
.ex-btn:disabled {
    background-color: gray;
}

We can see the changes are abrupt and not very appealing. Now, let's try to create it with transitions:

.ex-btn {
        background-color: blue;
        transition: background-color 0.25s ease-in-out;
    }
    .ex-btn:hover {
        background-color: darkblue;
    }
    .ex-btn:active {
        background-color: yellow;
    }
    .ex-btn:disabled {
        background-color: gray;
    }

Much better, right? The transitions make the changes smoother and more appealing to the users.

2. Animations

What does the W3C say? 🧐

CSS allows animation of HTML elements without using JavaScript!.

Let us ponder in that affirmation for a second here. Back in the day -I guess, I wasn't around really- if a developer wanted to animate an element, they'd have to use JavaScript. But now, thanks to the advancements in CSS, we can animate elements without having to write a single line of JS code. This has several implications, but the most important one is that these animations are pre-calculated by the browser, which makes them more efficient. This comes with a cost, though: CSS animations are less flexible than JavaScript animations, but there are ways to work around that.


How can we use them?

Let's take a look at the main properties that take part in an animation. We can distinguish two main parts: Keyframes and Animation Properties.

1. Keyframes

Keyframes are the steps of the animation. They define the start and end of the animation, as well as any intermediate steps. We can define as many keyframes as we want, but the most common ones are 0% and 100%, which represent the start and end of the animation, respectively. Let's see an example: 

@keyframes example-1 {
    0% {
        background-color: yellow;
    }
    100% {
        background-color: orange;
    }
}

In this example, we have defined a keyframe of name example-1 that changes the background color of the element from yellow to orange. Naming your animations is essential, as it's the way you'll reference them later on.

As we said, we can define as many keyframes as we want. Let's see another example with some intermediate steps in the animation: 

@keyframes example-2 {
    0% {
        background-color: red;
    }
    15% {
        background-color: orange;
    }
    30% {
        background-color: yellow;
    }
    45% {
        background-color: green;
    }
    60% {
        background-color: blue;
    }
    75% {
        background-color: indigo;
    }
    100% {
        background-color: violet;
    }
}

But of course, we can change any property we want, not only the background color.

@keyframes example-3 {
    0% {
        scale: 0.9;
    }
    100% {
        scale: 1.1;
    }
}

💙

2. Animation Properties

Similar to transitions, animations have several properties that define how the animation will behave. Let's take a look at the ones that differ from transitions:

2.1. Animation Name

We have already seen this one. animation-name is the property that references the keyframes we want to use in the animation. For example: 

.element {
    animation-name: example-1;
}
2.2. Animation Iteration Count

This property specifies how many times the animation will run. For example, if we want the animation to run indefinitely, we'd write: 

.element {
    animation-iteration-count: infinite;
}

It can also be a number, like 5, if we want the animation to run 5 times.

2.3. Animation Direction

This property specifies the direction of the animation. For example, if we want the animation to run in reverse, we'd write: 

.element {
    animation-direction: reverse;
}

Other options are normal, alternate, and alternate-reverse. These last two options make the animation run in reverse every other iteration, like the previous example with the beating heart 💙.

2.4. Animation Fill Mode

This property specifies what happens before and after the animation. For example, if we want the element to maintain the properties of the last keyframe after the animation has finished, we'd write: 

.element {
    animation-fill-mode: forwards;
}

Other options are none, backwards, and both. These options make the element maintain the properties of the first keyframe before the animation starts, or both, respectively.

2.5 Other

In the same way as with transitions, we can also specify the duration, timing-function, and delay of the animation.

Now, back to compacting!

Instead of writing all of these properties separately, we can write them all in one line, just like in transitions: 

.element {
    animation: example-1 2s infinite alternate forwards ease-in-out;
}

On to some more examples!

Let's take a look at some more (and cooler) examples of animations:

1. Loading Spinner

Let's begin by creating something easy but really useful: a loading spinner. We can create a spinner with just a few lines of CSS: 

@keyframes spinner {
    0% {
        transform: rotate(0deg);
    }
    100% {
        transform: rotate(360deg);
    }
}
.spinner {
    animation: spinner 1s infinite linear;
}

Now, imagine the amount of JS code you'd have to write to achieve the same effect. CSS animations are really a timesaver when it comes to simple, reusable animations.

2. Button Animation on Hover

As with transitions, we can also create animations on hover. Let's see an example of a button: 

@keyframes button-hover {
    0%, 100% {
        transform: scale(1);
    }
    50% {
        transform: scale(1.1);
    }
}
.button-hover:hover {
    animation: button-hover 0.5s infinite;
}

What's special about this is that, as we want 0% and 100% to be the same, we can write them together.

3. Card Animation

Let's kick it up a notch now. Let's create a card that both appears from the right on render and, when hovered, does a little dance. We can achieve this with the following code: 

.slide-in-animation {
    animation: slide-in 1s ease-in-out;
}

.card-dance-animation {
    animation: card-dance 0.25s 3 ease-in-out;
}

@keyframes slide-in {
    0% {
        transform: translateX(100%);
        opacity: 0;
    }
    100% {
        transform: translateX(0);
        opacity: 1;
    }
}

@keyframes card-dance {
    0% {
        transform: rotate(0deg);
    }
    25% {
        transform: rotate(5deg);
    }
    50% {
        transform: rotate(0deg);
    }
    75% {
        transform: rotate(-5deg);
    }
    100% {
        transform: rotate(0deg);
    }
}

When do we use them?

CSS animations are great to add a little bit of ✨ flare ✨ to your website. Most of the times, they aren't really a necessity, but they can make your website look more professional and engaging, as well as help it look cohesive and consistent.


One of the best uses of animations is to conceal long loading times. Apple, for example, knows this very well and uses several kinds of animations and gimmicks to trick the brain of the user into thinking that their device is faster than it actually is. This is a great way to improve user experience and make the user feel more comfortable with your website.


If we take a look at the card grid example above, we can see that the cards take a total of 2.25 seconds to completely be displayed to the user. This well be plenty of time for us to load any data we need to display, making the waiting time not be as noticeable to the final user.
